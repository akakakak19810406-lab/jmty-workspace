# JMTY Post Rewrite Rules

## Purpose

- ジモティー（Jimoty）の求人投稿用として、読者が公式LINEに登録したくなるような魅力的で詳細な文章を作成する。
- 給与面を最大のセールスポイントとして強調し、求職者の目を引く構成にする。
- 親しみやすさと企業・事業所としての信頼感を両立させたトーンを維持する。
- GEMや外部GEMプロンプトは使わず、このファイルの指示を正本にしてリライトする。

## Output Format

- 出力はコードブロックで行う。
- 表は使わず、すべてテキストのみで構成する。
- `**` や `##` などのMarkdown装飾記号は使わない。
- スプレッドシートへのエクスポートを想定した特殊な書式は使わない。
- JMTY投稿フォームに貼り付けるときは、コードブロックの囲み記号を除いた本文だけを使う。

## Title Rules

- タイトルは金額、職種、魅力的な一言を端的に入れる。
- 給与額、手当、月収などの金銭的メリットをメインに据える。
- 毎回切り口を変え、別の記事に見える独自の視点を入れる。
- 冒頭で職種の魅力が伝わるようにする。

## Body Structure

本文は次の順序を基本にする。

1. 違和感フック
2. 共感
3. 問題提起
4. 再定義
5. 作業
6. 体感ストーリー
7. やること
8. 再現性
9. 楽さ
10. 失敗回避
11. 環境
12. 緊急性
13. CTA

## Writing Style

- 安心させる前に違和感で止める。
- 説明ではなく体感させる。
- `理解`ではなく`想像させる`。
- 本文は段落を分け、一塊が長くなりすぎないようにする。
- 具体的で詳細な、ボリュームのある長文にする。
- 絵文字を多めに使い、視覚的に明るく親しみやすい印象にする。
- 親しみやすさはありつつ、企業・事業所としてのマナーを守る。
- `めちゃくちゃ楽しみにしています` や `ワクワク` のような過剰で子供っぽい表現は禁止。

## Required Content

- 要項（勤務時間、場所、給与詳細など）は必ず箇条書きでまとめる。
- 公式LINEへの誘導文を本文内に合計3回、自然な流れで入れる。
- 指定されたURLや `【公式LINEURL】` は完全一致で残す。
- 最後の一文には、`お会いできるのを楽しみにしています` または、その時々に合わせた一押しのニュアンスを入れる。
- 最後の挨拶は毎回バリエーションを変える。

## Remote / Nationwide Expression Rules

- 全国どこで投稿されても合う文章にする。
- `満員電車` という表現は使わない。
- `通勤時間が大変` という内容は毎回入れない。入れる版と入れない版をランダムに分ける。
- 入れる場合は、全国対応しやすい表現として `通勤時間にAIへ指示を出すだけ` のような軽い表現にする。
- 地域を限定する言い回し、駅名、都道府県名、市区町村名は、在宅本文には入れない。

## Final Check

- 別の記事に見える独自の視点があるか。
- 読者が違和感で目を止めるか。
- 最後まで読ませる引力があるか。
- 文末パーツの挨拶バリエーションが入っているか。
- 指定URLや `【公式LINEURL】` が完全一致しているか。
- 給与、職種、勤務条件、雇用形態など、元投稿の重要条件を変えていないか。
