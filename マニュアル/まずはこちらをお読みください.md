# JMTY Workspace 概要説明マニュアル

このファイルは、JMTY workspace の入口マニュアルです。
ジモティ投稿文の作成、入力素材の管理、生成済み投稿文の保存、JMTY 用の agent / slash command 運用だけを扱います。

## 最初に確認すること

1. このリポジトリのルートにいることを確認します。

```bash
pwd
ls AGENTS.md RULES.md README.md
```

2. `JMTY_ROOT` を登録します。

```bash
python3 .agent/skills/common/scripts/jmty_runtime.py setup-local-machine --repo-root .
```

3. 設定された場所を確認します。

```bash
echo "$JMTY_ROOT"
```

4. slash command adapter を同期します。

```bash
python3 scripts/sync_cross_cli_commands.py
```

## JMTY 投稿作成

`/jmty` は `.agent/skills/jmty/jmty-posts/SKILL.md` を入口にします。

- 工場求人: `jmty-posts-factory-14`
- 在宅求人: `jmty-posts-remote-14`
- 14パターン生成: `jmty-posts-14-variants`
- Google Drive 同期: `jmty-posts-gdrive-sync`

入力素材は `inputs/jmty_factory_cases/` または `inputs/jmty_remote_samples/` に置きます。
生成済み投稿文は `outputs/jmty/factory/` または `outputs/jmty/remote/` に保存します。

## Git 運用

Git 操作は `.agent/skills/common/git-workflow/SKILL.md` を正本にします。

```bash
git -C "$JMTY_ROOT" status --short
git -C "$JMTY_ROOT" fetch origin
```

`/git` は Discord 報告あり、`/git-nd` は Discord 報告なしです。
秘密情報は `config/discord-jmty-webhook.json` に置き、Git へ含めません。

## 使う slash command

| コマンド | 用途 |
|---|---|
| `/setup` | JMTY workspace の状態確認 |
| `/pull` | 最新変更の取り込み |
| `/c` | コミットのみ |
| `/git` | リモート反映と必要に応じた Discord 報告 |
| `/git-nd` | リモート反映のみ |
| `/jmty` | ジモティ投稿文作成 |
| `/jmty-weekly` | ジモティ週次素材処理 |
| `/ceo` | 適切なスキルへの振り分け |

## 注意

- JMTY と関係しない動画制作、Web 制作、note 記事制作の素材はこの repo に混ぜません。
- 個人作業は `personal/<account>/` 配下に置きます。
- API キー、Webhook URL、認証情報はコミットしません。
