# JMTY GUI

ジモティー週次作業をブラウザで操作するローカルGUIです。
データベースは使わず、既存の `inputs/` と `outputs/` のファイルを読み書きします。

## 起動

```bash
python3 scripts/jmty_gui.py --open
```

標準のURLは次です。

```text
http://127.0.0.1:8787/
```

ポートを変える場合:

```bash
python3 scripts/jmty_gui.py --port 8790 --open
```

## できること

- アカウントごとに `工場` / `在宅1` / `在宅2` の地域、投稿文、画像状態を一覧表示する
- `jmty_weekly_assets.py prepare` を `投稿文作成` ボタンから実行する
- `rotate-sheet --dry-run` を `地域確認` ボタンから実行する
- `rotate-sheet` を `地域変更` ボタンから実行する
- `sync-drive --purge-account-images` を `反映` ボタンから実行する
- `validate-output` を `検証` ボタンから実行する
- `inputs/jmty_image_prompt_templates/` の画風プロンプトテンプレートを登録・編集する
- 画風テンプレートの架空条件サンプル画像を生成し、一覧サムネイルにする
- アカウント別に `工場` / `在宅1` / `在宅2` の画像生成を開始する
- 画像と投稿文の内容が合っているかを、1件ずつまたは一括で検証する
- 必要に応じて生成依頼ファイルを `outputs/jmty-gui/generation_requests/` に作る
- 生成済み画像を `outputs/jmty-weekly/current/<アカウント名>/` に取り込む
- 画像を確認してOK状態にする
- 投稿文をGUI上で編集して保存する
- `シート読込` で `アカウント情報` シートの最新データを読み込む
- メイン画面で、どのアカウントがどの地域でどんな投稿文かを確認する
- `基本情報設定` で列名・列文字・行番号・セル位置を確認する
- `基本情報設定` で列設定を変更し、`outputs/jmty-gui/sheet_mapping.json` に保存する
- アカウントごとのシート情報をGUIで編集し、変更セルだけGoogle Sheetsへ書き戻す

## 画像生成の扱い

各アカウントの `工場` / `在宅1` / `在宅2` 欄に `画像生成` ボタンがあります。
ボタンを押すと、GUIが投稿文と地域を読み取り、登録済みの `画風テンプレ` から合うものを自動選択して画像プロンプトを作ります。
その後、ローカルの Codex CLI / App Server 側のログイン状態を使って `codex exec` をバックグラウンド起動し、Codexの画像生成で求人バナーを作ります。

生成中は対象カード内に進捗バーと現在の状態が表示されます。
生成が終わると、正方形プレビューに画像が表示されます。
画像が良ければ `OK`、手元の画像を使う場合は `画像取込` を押します。

`画像検証` を押すと、対象画像と投稿文をCodexで照合します。
`画像一括検証` では、画像がある `工場` / `在宅1` / `在宅2` をまとめて確認します。
投稿文と月収、勤務条件、工場/在宅の種別が違う可能性があるものは赤く表示されます。
誤検出だった場合は `確認済みにする` を押すと赤表示を解除できます。
検証結果は `outputs/jmty-gui/image_validation.json` に保存されます。

保存先は次のようなアカウント別フォルダです。

```text
outputs/jmty-weekly/current/<アカウント名>/工場.jpg
outputs/jmty-weekly/current/<アカウント名>/在宅1.jpg
outputs/jmty-weekly/current/<アカウント名>/在宅2.jpg
```

直接生成ではなく依頼文だけ作りたい場合は、各欄の `依頼作成` を使います。
この場合は次の流れです。

1. GUIで `依頼作成` を押す
2. `outputs/jmty-gui/generation_requests/` に依頼ファイルが作られる
3. Codexでその依頼ファイルのプロンプトを使って画像を生成する
4. GUIの `画像取込` で生成画像を取り込む
5. プレビューで確認して `OK` を押す
6. `反映` でGoogle Driveとスプレッドシートへ反映する

## ファイルの置き場所

- GUI本体: `scripts/jmty_gui.py`
- 週次出力: `outputs/jmty-weekly/current/`
- GUI補助ファイル: `outputs/jmty-gui/`
- 画風テンプレート: `inputs/jmty_image_prompt_templates/`
- テンプレート一覧画像: `inputs/jmty_image_prompt_templates/_previews/`
- シート列設定: `outputs/jmty-gui/sheet_mapping.json`
- シート読込キャッシュ: `outputs/jmty-gui/sheet_cache.json`

## スプレッドシート操作

`シート読込` を押すと、Google Sheets の `アカウント情報` シートを読み直します。
読み込んだ内容はGUI上の `アカウント情報一覧` 欄に表示されます。
普段は、アカウント名、工場地域、工場投稿文、在宅地域、在宅1投稿文、在宅2投稿文だけを見ます。

初期の列設定は次です。

| 項目 | 初期列 |
|---|---|
| アカウントNo | A |
| アカウント名 | B |
| 工場地域 | H |
| 工場投稿文 | J |
| 在宅地域 | Q |
| 在宅1投稿文 | S |
| 在宅2投稿文 | U |

列構成が変わった場合は、`基本情報設定` を開いて列文字を直し、`列設定保存` を押します。
保存後に自動で最新データを読み直します。
列設定は普段触らない補助情報なので、メイン画面には常時表示しません。

アカウント行の `編集` を押すと、対象行の値を編集できます。
`シートへ保存` を押す前に、更新されるセルと変更前後の値が表示されます。
保存時は対象セルだけを `gws` 経由で更新します。

## 画風テンプレ操作

右側の `画風テンプレ` では、通常は登録済みテンプレ一覧だけを表示します。
新しく登録する場合は `新規テンプレ` を押します。
既存テンプレを直す場合は、テンプレ行の `編集` を押します。

テンプレ行の `見本生成` を押すと、架空の地域・月収・職種でサンプル画像を生成し、`inputs/jmty_image_prompt_templates/_previews/` に保存します。
保存後はテンプレ一覧のサムネイルとして表示されます。

新規登録時は次の動きです。

- `画像プロンプト` を入力した場合: その内容を保存し、架空条件で見本画像を生成する
- `見本画像` だけ入れた場合: 画像を参考に画風プロンプトを自動作成し、見本画像も生成する
- `一覧画像（手動上書き）` を入れた場合: その画像をサムネイルとして使い、自動生成はしない

## 注意

`地域変更`、`反映`、シート編集保存はGoogleスプレッドシートやGoogle Driveを更新します。
実行には `gws` が必要です。

## gws認証

`Access denied. No credentials provided.` と出る場合は、`gws` のGoogleログインが未完了です。
GUIはファイル保存型の認証情報を使うため、次でログインします。

```bash
GOOGLE_WORKSPACE_CLI_KEYRING_BACKEND=file gws auth login
```

ブラウザでGoogleログインが終わったら、GUIの `シート読込` を押し直します。
